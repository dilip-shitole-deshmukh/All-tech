# 🧠 1️⃣ Numbers

## 🔹 Definition

`Number` is a primitive data type in JavaScript used to represent both **integers and floating-point values**.

JavaScript uses **IEEE-754 64-bit floating point** format.

---

## 🔹 Deep Explanation

### 📌 Types of Numbers in JS

- Integer → `10`
- Float → `10.5`
- Scientific → `1e6`
- Special values:
  - `NaN`
  - `Infinity`
  - `-Infinity`

### 📌 JS has only ONE numeric type

Unlike Java/C#, JS does NOT have:

- int
- float
- double
  Everything is `Number` (except `BigInt`)

---

### 📌 Internal Representation (IEEE-754)

```
0.1 + 0.2 !== 0.3
```

Because:

- Binary floating point precision issue
- 0.1 and 0.2 cannot be stored exactly

---

### 📌 Special Numeric Values

| Value                     | Meaning                                |
| ------------------------- | -------------------------------------- |
| `NaN`                     | Not a number (but typeof === "number") |
| `Infinity`                | Overflow                               |
| `Number.MAX_SAFE_INTEGER` | 2^53 - 1                               |
| `Number.MIN_SAFE_INTEGER` | -(2^53 - 1)                            |

---

## 🔹 Examples

```js
let a = 10;
let b = 0.1 + 0.2;

console.log(b); // 0.30000000000000004
console.log(Number.isNaN(NaN)); // true
console.log(Number.isFinite(100)); // true
```

BigInt:

```js
let big = 9007199254740991n;
```

---

## 🔹 Real-World Use Cases

✔️ Price calculation
✔️ Pagination indexes
✔️ Form validations
✔️ Animation timing
✔️ Currency calculation (with care!)

---

## 🔹 When to Use?

| Situation            | Use             |
| -------------------- | --------------- |
| Small numeric values | Number          |
| Financial system     | Decimal library |
| Large IDs            | BigInt          |
| Counters             | Number          |

---

## ✅ Advantages

✔️ Simple
✔️ Built-in math methods
✔️ Fast

---

## ⚡ Pitfalls

❌ Floating precision errors
❌ NaN propagation
❌ Implicit type coercion

---

## 🔹 Senior Best Practices

✔️ Use `Number.isNaN()` instead of `isNaN()`
✔️ Use `===` not `==`
✔️ For money → use libraries like `decimal.js`
✔️ Validate before arithmetic

# 🧵 2️⃣ Strings

## 🔹 Definition

`String` is a primitive data type representing text.

Strings in JS are **immutable**.

---

## 🔹 Deep Explanation

### 📌 Ways to create

```js
let a = "Hello";
let b = "Hello";
let c = `Hello ${name}`;
```

### 📌 Immutable Nature

```js
let str = "Hello";
str[0] = "h";
console.log(str); // still "Hello"
```

It creates new string internally.

---

### 📌 String Internals

- Stored as UTF-16
- Indexed access allowed
- Methods return new string

---

## 🔹 Examples

```js
let name = "Dilip";
console.log(name.length);
console.log(name.toUpperCase());
console.log(name.includes("lip"));
```

Template literal:

```js
let message = `Welcome ${name}`;
```

---

## 🔹 Real-World Use Cases

✔️ UI rendering
✔️ API data
✔️ URL building
✔️ Logging
✔️ Form inputs

---

## 🔹 When to Use?

| Situation    | Use               |
| ------------ | ----------------- |
| Dynamic text | Template literals |
| Multi-line   | Backticks         |
| Localization | i18n library      |

---

## ✅ Advantages

✔️ Easy manipulation
✔️ Template literals powerful
✔️ Immutable → safe

---

## ⚡ Pitfalls

❌ Heavy string concatenation → performance
❌ Unicode issues
❌ XSS risk in frontend

---

## 🔹 Senior Best Practices

✔️ Use template literals
✔️ Sanitize user input
✔️ Avoid repeated concatenation in loops
✔️ Use `.trim()` before validation

# 🔘 3️⃣ Booleans

## 🔹 Definition

Represents logical values: `true` or `false`.

---

## 🔹 Deep Explanation

### 📌 Truthy & Falsy

Falsy values:

```
false
0
-0
""
null
undefined
NaN
```

Everything else → truthy

---

## 🔹 Examples

```js
if ("") {
  console.log("Won’t run");
}

if ("hello") {
  console.log("Runs");
}
```

Boolean conversion:

```js
Boolean(0); // false
!!"text"; // true
```

## ✅ Advantages

✔️ Avoid relying on implicit truthiness

# 📦 4️⃣ Arrays

## 🔹 Definition

Ordered collection of values (can store mixed types).

---

## 🔹 Deep Explanation

### 📌 Characteristics

- Zero-indexed
- Dynamic size
- Reference type
- Mutable

---

### 📌 Internal

JS arrays are:

- Objects under the hood
- Indexed keys
- Optimized for dense arrays

---

## 🔹 Examples

```js
let arr = [1, 2, 3];

arr.push(4);
arr.map((n) => n * 2);
arr.filter((n) => n > 2);
arr.reduce((a, b) => a + b, 0);
```

---

## 🔹 Real-World Use Cases

✔️ Rendering lists in React
✔️ API data collection
✔️ Form fields group
✔️ Data transformation

---

## 🔹 When to Use?

| Situation    | Use        |
| ------------ | ---------- |
| Ordered data | Array      |
| Unique items | Set        |
| Key-value    | Object/Map |

---

## ✅ Advantages

✔️ Powerful methods
✔️ Functional programming support

---

## ⚡ Pitfalls

❌ Mutation bugs
❌ Sparse arrays
❌ O(n) search

---

## 🔹 Senior Best Practices

✔️ Prefer immutable operations
✔️ Avoid direct mutation in React
✔️ Use `map/filter/reduce`
✔️ Use `Array.isArray()`

# 🏗 5️⃣ Objects

## 🔹 Definition

Collection of key-value pairs.

Objects are **reference types**.

---

## 🔹 Deep Explanation

### 📌 Key rules

- Keys are strings or symbols
- Not truly ordered (but ES6 preserves insertion order)
- Stored in heap

---

## 🔹 Examples

```js
let user = {
  name: "Dilip",
  age: 30,
};

user.email = "dilip@gmail.com";
delete user.age;
```

Access:

```js
user.name;
user["name"];
```

Destructuring:

```js
const { name } = user;
```

---

## 🔹 Real-World Use Cases

✔️ API responses
✔️ Redux state
✔️ Config objects
✔️ Entity models

---

## 🔹 When to Use?

| Situation       | Use    |
| --------------- | ------ |
| Structured data | Object |
| Dynamic keys    | Map    |
| JSON            | Object |

---

## ✅ Advantages

✔️ Flexible
✔️ Fast key lookup O(1)

---

## ⚡ Pitfalls

❌ Shallow copy issues
❌ Reference mutation
❌ Prototype pollution

---

## 🔹 Senior Best Practices

✔️ Use spread for cloning
✔️ Deep clone carefully
✔️ Freeze objects if needed
✔️ Use optional chaining `?.`

# 🎯 15 Senior-Level Interview Q&A

---

### 1️⃣ Why is `typeof null` "object"?

Historical JS bug.
Null was implemented as a null pointer object reference.

---

### 2️⃣ Difference between primitive and reference types?

Primitive → stored in stack
Reference → stored in heap

Copy behavior differs.

---

### 3️⃣ Why 0.1 + 0.2 !== 0.3?

Floating-point precision issue (IEEE-754).

---

### 4️⃣ Difference between `==` and `===`?

`==` → type coercion
`===` → strict equality

Senior rule: Always use `===`

---

### 5️⃣ How to check if variable is array?

```js
Array.isArray(arr);
```

---

### 6️⃣ How to clone object?

```js
const copy = { ...obj };
```

Deep clone:

```js
structuredClone(obj);
```

---

### 7️⃣ What is NaN?

Special numeric value meaning invalid number.
NaN !== NaN.

---

### 8️⃣ What are truthy/falsy values?

Falsy: `false, 0, "", null, undefined, NaN`

---

### 9️⃣ Difference between Object and Map?

| Object                 | Map                     |
| ---------------------- | ----------------------- |
| String keys            | Any key type            |
| Less predictable order | Maintains order         |
| Faster simple cases    | Better for dynamic keys |

---

### 🔟 Why strings are immutable?

To improve:

- Performance
- Memory reuse
- Predictability

---

### 1️⃣1️⃣ What happens when you assign object to another variable?

It copies reference, not value.

---

### 1️⃣2️⃣ How to prevent object mutation?

```js
Object.freeze(obj);
```

---

### 1️⃣3️⃣ What is sparse array?

```js
let arr = [];
arr[10] = 5;
```

Creates empty slots.

---

### 1️⃣4️⃣ How does JS handle large numbers?

Use `BigInt`.

---

### 1️⃣5️⃣ How are arrays different from objects internally?

Arrays are specialized objects optimized for numeric keys and sequential memory.
