# 🔹 Definition

Operators are special symbols that perform operations on values (operands).

In JS, operators are:

- Type-coercion aware
- Overloaded (e.g., `+` works for numbers & strings)
- Short-circuit capable (logical operators)

---

# 1️⃣ Arithmetic Operators ➕

---

## 🔹 Definition

Used to perform mathematical operations.

### Core Arithmetic Operators:

| Operator | Meaning        |
| -------- | -------------- |
| `+`      | Addition       |
| `-`      | Subtraction    |
| `*`      | Multiplication |
| `/`      | Division       |
| `%`      | Modulus        |
| `**`     | Exponent       |
| `++`     | Increment      |
| `--`     | Decrement      |

---

## 🔥 Deep Explanation

### 📌 `+` is overloaded

```js
5 + 3; // 8
"5" + 3; // "53"
```

If one operand is string → string concatenation.

---

### 📌 Division by Zero

```js
10 / 0; // Infinity
```

JS does NOT throw error.

---

### 📌 Modulus `%`

Returns remainder:

```js
10 % 3; // 1
```

Important for:

- Pagination
- Even/odd check

---

### 📌 Exponent `**`

```js
2 ** 3; // 8
```

---

### 📌 Pre vs Post Increment

```js
let x = 5;

console.log(x++); // 5 (then becomes 6)
console.log(++x); // 7
```

---

## 🔹 Examples

```js
let price = 100;
let tax = 0.18;

let total = price + price * tax;
```

---

## 🌍 Real-World Use Cases

✔️ E-commerce calculations
✔️ Animation timing
✔️ Pagination logic
✔️ Game logic
✔️ Financial systems

---

## ⚡ Pitfalls

❌ Floating precision:

```js
0.1 + 0.2; // 0.30000000000000004
```

❌ String coercion bugs

# 2️⃣ Comparison Operators ⚖️

---

## 🔹 Definition

Used to compare values and return boolean.

| Operator | Meaning           |
| -------- | ----------------- |
| `==`     | Loose equality    |
| `===`    | Strict equality   |
| `!=`     | Loose inequality  |
| `!==`    | Strict inequality |
| `>`      | Greater           |
| `<`      | Less              |
| `>=`     | Greater or equal  |
| `<=`     | Less or equal     |

---

## 🔥 Deep Explanation

### 📌 Strict vs Loose

```js
5 == "5"; // true
5 === "5"; // false
```

Always prefer `===`.

---

### 📌 Object Comparison

```js
{} === {} // false
```

Reference comparison.

---

### 📌 String Comparison

```js
"apple" < "banana"; // true
```

Lexicographic comparison (Unicode-based).

---

### 📌 Comparing null & undefined

```js
null == undefined; // true
null === undefined; // false
```

# 3️⃣ Logical Operators 🔀

---

## 🔹 Definition

Used for logical conditions.

| Operator | Meaning            |     |     |
| -------- | ------------------ | --- | --- |
| `&&`     | AND                |     |     |
| `        |                    | `   | OR  |
| `!`      | NOT                |     |     |
| `??`     | Nullish coalescing |     |     |

---

## 🔥 Deep Explanation

### 📌 `&&` (AND)

Returns first falsy value.

```js
true && "Hello"; // "Hello"
false && "Hi"; // false
```

---

### 📌 `||` (OR)

Returns first truthy value.

```js
"" || "Default"; // "Default"
```

---

### 📌 Short-Circuiting

```js
isLoggedIn && showDashboard();
```

If false → second expression not executed.

---

### 📌 Nullish Coalescing `??`

```js
0 ?? 100; // 0
```

Different from `||`.

```js
0 || 100; // 100
```

Because `||` treats 0 as falsy.

---

### 📌 NOT Operator

```js
!true; // false
!!"text"; // true
```

Used for boolean conversion.

---

## 🌍 Real-World Use Cases

✔️ React conditional rendering
✔️ Default values
✔️ Auth guards
✔️ Feature flags
✔️ Optional props

# 🎓 15+ Senior-Level Interview Q&A

---

### 1️⃣ Why is `+` operator tricky?

Because it performs both addition and string concatenation.

---

### 2️⃣ Difference between `||` and `??`?

`||` checks truthy, `??` checks null/undefined only.

---

### 3️⃣ What is short-circuiting?

Evaluation stops once result is determined.

---

### 4️⃣ Why prefer `===`?

Avoid coercion bugs.

---

### 5️⃣ Why `NaN === NaN` is false?

NaN is not equal to itself.

---

### 6️⃣ How to check NaN correctly?

`Number.isNaN(value)`

---

### 7️⃣ Why `0 || 10` returns 10?

Because 0 is falsy.

---

### 8️⃣ What does `!!value` do?

Converts to boolean.

---

### 9️⃣ What happens in `true + true`?

`2` (true → 1)

---

### 🔟 What is `%` used for?

Remainder calculation (even/odd check).

---

### 1️⃣1️⃣ Why `10 / 0` doesn't throw?

JS returns Infinity.

---

### 1️⃣2️⃣ Why `{}` === `{}` false?

Different references.

---

### 1️⃣3️⃣ What does `&&` return?

Last evaluated truthy value or first falsy.

---

### 1️⃣4️⃣ How logical operators used in React?

Conditional rendering:

```js
isAdmin && <AdminPanel />;
```

---

### 1️⃣5️⃣ What is operator precedence?

Order in which operators are evaluated.

Example:

```js
2 + 3 * 4; // 14
```
