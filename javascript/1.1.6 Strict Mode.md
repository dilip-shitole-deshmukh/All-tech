# 🧠 Strict Mode in JavaScript

---

# 🔹 Definition

**Strict Mode** is a restricted variant of JavaScript that:

- Eliminates silent errors
- Prevents unsafe actions
- Disallows certain bad practices
- Makes code more secure and predictable

It is enabled using:

```js
"use strict";
```

---

# 🔥 Why Strict Mode Exists?

Old JavaScript (ES3/ES5 non-strict):

- Allowed dangerous patterns
- Silently ignored mistakes
- Caused global leaks
- Made debugging hard

Strict mode was introduced in **ES5 (2009)** to fix this.

---

# 🔹 How to Enable Strict Mode

### 1️⃣ Entire Script

```js
"use strict";

x = 10; // ❌ Error
```

### 2️⃣ Inside Function

```js
function test() {
  "use strict";
}
```

### 3️⃣ ES Modules

⚠️ Important:

👉 ES Modules are automatically in strict mode.

```js
// module.js
export const a = 10; // strict by default
```

---

# 🔥 Deep Explanation (Every Rule Explained)

---

# 1️⃣ Prevents Accidental Globals

❌ Non-strict:

```js
function test() {
  x = 10; // creates global variable
}
```

✅ Strict:

```js
"use strict";

function test() {
  x = 10; // ❌ ReferenceError
}
```

🧠 Why Important?
Accidental globals cause memory leaks and unpredictable bugs.

---

# 2️⃣ `this` Behavior Changes

Non-strict:

```js
function show() {
  console.log(this);
}
show(); // Window (browser)
```

Strict:

```js
"use strict";

function show() {
  console.log(this);
}
show(); // undefined
```

🧠 Why?
Prevents unintended global object binding.

---

# 3️⃣ Duplicate Parameters Not Allowed

❌ Non-strict:

```js
function sum(a, a) {}
```

✅ Strict:

```js
"use strict";
function sum(a, a) {} // ❌ SyntaxError
```

---

# 4️⃣ Deleting Variables Not Allowed

```js
"use strict";

let x = 10;
delete x; // ❌ Error
```

---

# 5️⃣ Octal Literals Not Allowed

❌

```js
"use strict";
let x = 010; // ❌
```

---

# 6️⃣ Assigning to Read-Only Properties Throws Error

```js
"use strict";

const obj = Object.freeze({ a: 1 });
obj.a = 2; // ❌ TypeError
```

Non-strict → silently fails.

---

# 7️⃣ `arguments` Object Restrictions

In strict mode:

```js
"use strict";

function test(a) {
  arguments[0] = 100;
  console.log(a); // still original value
}
```

No automatic sync between parameters and arguments.

---

# 8️⃣ eval() is Safer

Non-strict:

```js
eval("var x = 10;");
console.log(x); // accessible
```

Strict:

```js
"use strict";
eval("var x = 10;");
console.log(x); // ❌ Not defined
```

Prevents scope pollution.

# 🎓 15+ Interview Q&A

---

### 1️⃣ What is strict mode?

Restricted version of JS that catches common mistakes.

---

### 2️⃣ How do you enable strict mode?

`"use strict";` at top of file or function.

---

### 3️⃣ Are ES modules strict by default?

Yes.

---

### 4️⃣ What happens to `this` in strict mode?

It becomes `undefined` in regular function calls.

---

### 5️⃣ Why is that safer?

Prevents unintended global object access.

---

### 6️⃣ What errors does strict mode catch?

- Undeclared variables
- Duplicate parameters
- Deleting variables
- Invalid assignments

---

### 7️⃣ Can strict mode improve performance?

Yes. Engines can optimize better due to predictable behavior.

---

### 8️⃣ Does strict mode affect arrow functions?

Arrow functions already don’t bind their own `this`.

---

### 9️⃣ What happens with eval in strict mode?

It cannot create variables in outer scope.

---

### 🔟 Is strict mode required in modern apps?

If using ES modules → automatically strict.

---

### 1️⃣1️⃣ Does strict mode change variable hoisting?

No, but prevents unsafe usage.

---

### 1️⃣2️⃣ Can strict mode be applied partially?

Yes, per function.

---

### 1️⃣3️⃣ Why do frameworks prefer strict mode?

Cleaner debugging and safer execution.

---

### 1️⃣4️⃣ What happens if you assign to a non-writable property?

Throws TypeError (instead of silent fail).

---

### 1️⃣5️⃣ Is strict mode backward compatible?

Yes, but may break old unsafe code.
