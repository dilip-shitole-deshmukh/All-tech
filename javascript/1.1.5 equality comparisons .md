# 🧠 `==` vs `===` vs `typeof` — Equality Comparisons in JavaScript

---

# 🔹 Definition

### `==` → Loose Equality

Compares values **after type coercion**

### `===` → Strict Equality

Compares values **without type coercion**

### `typeof` → Type Operator

Returns the type of a variable as a string

---

# 🔥 1️⃣ `==` (Loose Equality)

## 🔹 Deep Explanation

When you use:

```js
a == b;
```

JavaScript follows the **Abstract Equality Comparison Algorithm** (ECMAScript spec).

It:

1. Checks if types are same → compare normally
2. If types differ → applies coercion rules

---

## 🔥 Type Coercion Rules (Critical for Interview)

### 📌 String & Number

```js
"5" == 5; // true
```

Because:

- "5" → converted to number 5

---

### 📌 Boolean Comparison

```js
true == 1; // true
false == 0; // true
```

Because:

- true → 1
- false → 0

---

### 📌 null & undefined

```js
null == undefined; // true
```

But:

```js
null === undefined; // false
```

---

### 📌 Object vs Primitive

```js
[1] == "1"; // true
```

Why?

- `[1]` → converted to string `"1"`

---

### 📌 Dangerous Cases

```js
0 == ""         // true
0 == false      // true
"" == false     // true
[] == false     // true
```

These are **type coercion traps**.

---

## 🔹 Example

```js
console.log("0" == 0); // true
console.log(false == 0); // true
```

---

## ⚡ Why It’s Dangerous?

Because coercion can lead to unpredictable logic.

---

# 🔥 2️⃣ `===` (Strict Equality)

## 🔹 Deep Explanation

```js
a === b;
```

Rules:

- No type conversion
- Same type?
- Same value?

Only then → true.

---

## 🔹 Examples

```js
5 === 5; // true
"5" === 5; // false
true === 1; // false
null === null; // true
```

---

## 🔥 Special Case: NaN

```js
NaN === NaN; // false
```

Because:

- NaN is not equal to anything (even itself)

Correct way:

```js
Number.isNaN(NaN); // true
```

---

## 🔥 Objects Comparison

```js
{} === {} // false
```

Why?

- Different references in memory

```js
const obj = {};
obj === obj; // true
```

Reference comparison.

---

# 🔥 3️⃣ `typeof`

## 🔹 Definition

Returns a string representing the type.

---

## 🔹 Examples

```js
typeof 10; // "number"
typeof "hello"; // "string"
typeof true; // "boolean"
typeof undefined; // "undefined"
typeof {}; // "object"
typeof []; // "object"
typeof null; // "object" ❗
```

---

## 🔥 Famous Interview Question

Why is:

```js
typeof null === "object";
```

Answer:
Legacy bug from early JS implementation.

---

## 🔥 Better Type Checking

```js
Array.isArray(arr);
```

```js
value === null;
```

---

# 🔥 Internal Working (Senior Level)

### Strict Equality Algorithm:

1. If types different → false
2. If same type:
   - Primitive → compare value
   - Object → compare reference

---

### Loose Equality Algorithm:

- Converts types based on rules:
  - Boolean → Number
  - String → Number
  - Object → Primitive
  - null == undefined → true

# 🎯 When to Use? Why? Where? How?

---

## ✅ Use `===` When:

✔️ 99% of cases
✔️ Comparing primitives
✔️ Checking types
✔️ Business logic
✔️ Authentication

Example:

```js
if (role === "admin") {
}
```

---

## ⚠️ Use `==` When (Rare Case)

When you intentionally want:

```js
if (value == null) {
  // catches both null and undefined
}
```

This is a valid senior-level pattern.

---

## 🔍 Use `typeof` When:

✔️ Checking undefined variables safely

```js
if (typeof window !== "undefined") {
}
```

Important for SSR in React / Next.js.

# 🎓 15+ Senior-Level Interview Q&A

---

### 1️⃣ Difference between `==` and `===`?

`==` performs type coercion, `===` does not.

---

### 2️⃣ Why prefer `===`?

Predictable behavior.

---

### 3️⃣ Why is `NaN === NaN` false?

NaN is not equal to anything, including itself.

---

### 4️⃣ How to correctly check NaN?

`Number.isNaN(value)`

---

### 5️⃣ Why is `typeof null` "object"?

Legacy bug.

---

### 6️⃣ Why `[] == false` is true?

Because:

- [] → ""
- "" → 0
- false → 0

So 0 == 0 → true

---

### 7️⃣ Why `{}` === `{}` is false?

Different references.

---

### 8️⃣ What happens in `true == "1"`?

- true → 1
- "1" → 1
  → true

---

### 9️⃣ Is `===` faster than `==`?

Negligible difference in modern engines.

---

### 🔟 Why avoid implicit coercion?

It hides bugs.

---

### 1️⃣1️⃣ How to safely check null or undefined?

```js
if (value == null)
```

---

### 1️⃣2️⃣ Why `typeof` useful in SSR?

Avoid ReferenceError for undefined globals.

---

### 1️⃣3️⃣ How to check array?

`Array.isArray(arr)`

---

### 1️⃣4️⃣ How to compare deep objects?

Use:

- `lodash.isEqual`
- JSON.stringify (not recommended for complex)

---

### 1️⃣5️⃣ What is strict equality algorithm?

Checks type first, then value.
